# 워크플로우 문서화 스킬

완료된 작업의 워크플로우를 마크다운 문서로 기록합니다.

## 사용 시점

- 중요한 작업 단위가 완료되었을 때
- 복잡한 구현이나 리팩토링이 끝났을 때
- 버그 수정이나 문제 해결 세션이 종료되었을 때
- 사용자가 명시적으로 문서화를 요청할 때

## 문서화 프로세스

### 1단계: 정보 수집

대화 컨텍스트에서 다음 정보를 추출합니다:

- **사용자 요청**: 작업을 시작하게 된 원래 요청
- **실행 계획**: 계획 모드 출력 또는 수행된 전략적 접근 방식
- **수행 내용**: 모든 작업과 그 결과
- **변경 파일**: 생성, 수정, 삭제된 파일 목록
- **주요 결정**: 내린 결정과 그 근거
- **해결 과정**: 발생한 문제와 해결 방법

### 2단계: 토픽 식별

작업의 핵심 주제를 2-4개 키워드로 추출합니다:

```
예시:
- oauth_implementation
- database_migration
- bug_fix_memory_leak
- feature_user_authentication
```

- 영어 사용, 단어 사이 언더스코어
- 검색 가능하고 의미 있게 작성

### 3단계: 타임스탬프 획득

Bash 도구를 사용하여 현재 시간을 획득합니다:

```bash
# Windows (PowerShell)
powershell -Command "Get-Date -Format 'yyyyMMddHHmmss'"

# Unix/Linux/macOS
date +%Y%m%d%H%M%S
```

### 4단계: 파일명 생성

형식: `{YYYYMMDDHHMMSS}_{topic}.md`

예시: `20240115143022_authentication_system.md`

### 5단계: 문서 작성

다음 템플릿을 사용하여 마크다운 문서를 작성합니다:

```markdown
# 작업 기록: {topic}

## 작업 주제
{수행된 작업에 대한 명확하고 설명적인 제목}

## 요청 내용
{사용자의 원래 요청을 그대로 인용}

## 수행 계획
{실행 계획 - 계획 모드 출력 또는 전략적 접근 방식 재구성}

- 주요 목표
- 핵심 단계 또는 단계
- 기술적 접근 방식
- 사용된 도구 또는 방법

## 결과 요약
{포괄적인 요약}

- 달성한 내용
- 생성/수정된 파일과 간단한 설명
- 주요 기술적 결정과 그 이유
- 사용된 중요한 코드 패턴이나 접근 방식
- 발생한 문제와 적용된 해결책
- 현재 상태 및 다음 단계 (해당되는 경우)

## 메타데이터
- 작성일시: {YYYY-MM-DD HH:MM:SS}
- 관련 파일: {관련 파일 목록}
- 주요 키워드: {관련 태그 또는 키워드}
```

### 6단계: 파일 저장

Write 도구를 사용하여 문서를 저장합니다:

- 저장 경로: `docs/work-logs/`
- 디렉토리가 없으면 생성
- 생성된 파일명 사용

## 품질 기준

문서화는 다음을 만족해야 합니다:

| 기준 | 설명 |
|-----|------|
| 정확성 | 요청 및 달성 내용을 정확히 반영 |
| 완전성 | 모든 관련 정보를 누락 없이 포함 |
| 명확성 | 미래 독자가 이해할 수 있는 정확한 언어 사용 |
| 구조화 | 템플릿을 일관되게 따름 |
| 맥락성 | 작업에 익숙하지 않은 사람도 이해할 수 있는 배경 제공 |
| 간결성 | 철저하되 불필요한 장황함 회피 |

## 실행 지침

1. **확인 후 생성**: 문서화할 내용을 먼저 명시
2. **신중한 정보 추출**: 전체 대화 컨텍스트에서 정보 추출
3. **계획 재구성**: 명시적 계획이 없으면 수행된 작업을 분석하여 재구성
4. **결과 요약 충실**: 가장 가치 있는 섹션이므로 철저히 작성
5. **파일 생성 확인**: 저장 성공 여부를 사용자에게 보고

## 오류 처리

- 원래 요청을 파악할 수 없으면 사용자에게 명확히 요청
- 작업 컨텍스트가 불분명하면 추가 정보 요청
- 파일 생성 실패 시 오류를 명확히 보고하고 대안 제시
- `docs/work-logs/` 디렉토리 생성 불가 시 대체 경로 사용 후 사용자에게 알림

## 완료 후 출력

문서 생성 후:

1. 전체 경로와 함께 파일 생성 확인
2. 문서화된 내용의 간략한 요약 제공
3. 쉬운 참조를 위해 파일명 언급
